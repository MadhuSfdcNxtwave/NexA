  IMPORTANT SCOPE NOTE:
  - This table contains ONLY users with active learning platform access (i.e.,
    lp_status = 'ACTIVE'). Users whose access has expired or been revoked are

  Key distinction:
  - user_job_status is the top-level funnel stage (You Are Eligible, Applied,
    Did Not Apply, Not Interested, Hiring Completed, Requirements Closed).
  - user_job_application_status is only meaningful for users who applied and
    tracks granular hiring pipeline stages (resume screening, assessments,
    assignments, group discussion, telephonic, technical, HR rounds, Hired,
    Not Selected).

  Important considerations:
  - **Hiring-cycle time metrics use CALENDAR TIME, not working hours.** All
    `avg_days_*` measures spanning application -> process completion are
    computed as raw wall-clock day differences between datetimes. They do NOT
    exclude weekends, holidays, or non-working hours, and they do NOT apply any
    placements-team / company-side hiring business calendar. An application
    submitted on Friday and processed the following Tuesday will show 4 calendar
    days even though only ~2 are working days. Whenever you query this table
    for hiring-funnel SLA analysis (e.g. time-to-decision, time-to-hire), you
    MUST surface this caveat to the stakeholder so they interpret the numbers
    as calendar latency, not effective working-day latency.
